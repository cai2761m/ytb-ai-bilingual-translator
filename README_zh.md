# ytb-ai-bilingual-translator

## 关于

一个 Chrome 扩展（Manifest V3），为 YouTube 提供 **AI 驱动的中英双语字幕**。它自动获取 YouTube 视频的英文字幕轨道，将碎片化的字幕片段智能合并为自然的完整句子，通过可配置的 LLM API 翻译为中文（简体或繁体），并在视频播放器上直接渲染 **中英双语字幕叠加层** —— 同时隐藏 YouTube 原生的 CC 字幕窗口。

无需音频识别，扩展完全基于 YouTube 已有的字幕数据工作。

## 功能特性

- **句子感知字幕合并** —— 将 YouTube 词/短语级别的字幕片段重新组合为完整、可读的句子，并避免明显断句，例如 `following your / nose` 这类前后断裂。
- **AI 翻译** —— 使用 LLM API 进行上下文感知的字幕翻译，输出自然、高质量的结果。
- **优先级调度** —— 优先翻译当前播放位置附近的字幕，然后在后台继续翻译视频其余部分。
- **ASR 纠错** —— 可选启用 AI 纠正自动生成字幕中的常见语音识别错误。
- **可拖拽叠加层** —— 双语字幕叠加层支持长按拖拽重新定位，位置会被持久化保存。
- **可调节字体大小** —— 字幕大小可在设置页面中调整，范围为 0.7× 到 1.8×。
- **稳定翻译缓存** —— 在 `chrome.storage.local` 中缓存已翻译的字幕，并使用稳定的视频/字幕轨道指纹；刷新同一个视频时会复用已有翻译，避免重复调用 API。
- **处理中请求去重** —— 如果翻译尚未完成时刷新页面，同一个字幕 cue 的重复请求会尽量复用后台已有请求，减少重复消耗 token。
- **并行批量翻译** —— 支持同时发送多个翻译批次，按服务商使用不同批量/并发参数，并具备速率限制、坏 JSON、漏返回单条字幕、服务器错误等场景的退避和重试机制。
- **多翻译服务商支持**：
  - **DeepSeek**（默认）—— 只需 API Key 即可开箱即用。
  - **Gemini** —— Google Gemini API，具备专用的批处理参数。
  - **自定义 OpenAI 兼容 API** —— 任何实现了 OpenAI Chat Completions 接口的服务商，包括 API 中转站 / 模型号池服务。

## 安装

1. 打开 `chrome://extensions`。
2. 开启 **开发者模式**。
3. 点击 **"加载已解压的扩展程序"**。
4. 选择下载或克隆的项目文件夹。
5. 点击扩展图标打开设置页面，配置翻译 API。

## 使用方法

1. 打开任意带有英文字幕的 YouTube 视频。
2. 扩展会自动检测字幕轨道（优先使用自动生成的 `asr` 字幕，回退到手动字幕轨道）。
3. 双语字幕会随着翻译完成实时显示在视频上。
4. 如果未找到英文字幕轨道，会显示"未找到"状态。

## 配置说明

打开扩展的设置页面（点击扩展图标 → 设置）进行配置：

| 设置项 | 说明 |
|---|---|
| **翻译服务** | DeepSeek / Gemini / 自定义 OpenAI 兼容 API |
| **API Key** | 所选服务商的 API 密钥 |
| **Base URL** | API 端点地址（DeepSeek 和 Gemini 会自动填充） |
| **模型** | 模型名称（DeepSeek 和 Gemini 会自动填充） |
| **JSON 输出模式** | 请求结构化 JSON 输出（如果服务商不支持可关闭） |
| **ASR 纠错** | 开关 AI 语音识别纠错功能 |
| **源语言** | 英语 |
| **目标语言** | 简体中文 / 繁体中文 |
| **字幕大小** | 字幕缩放倍数（0.7× – 1.8×） |
| **启用字幕** | 开关叠加层显示，同时隐藏 YouTube 原生 CC 字幕 |

### 自定义 OpenAI 兼容 API

如果使用 OpenAI 兼容的 API 中转站，在设置页选择 **自定义 OpenAI 兼容 API**，然后填写：

| 字段 | 示例 |
|---|---|
| **API Key** | `sk-...` |
| **Base URL** | `https://anpin.ai/v1` |
| **模型** | `gpt-5.4` |

建议先保持 **JSON 输出模式** 开启。如果某个服务商不支持 OpenAI JSON mode，可以关闭该选项；扩展仍会尝试解析并恢复可用的翻译结果。

### 缓存行为

扩展会按视频、源字幕轨道身份、服务商、接口地址、模型、源语言、目标语言、ASR 纠错开关、字幕合并版本和缓存版本来缓存翻译结果。刷新同一个 YouTube 视频时，已成功翻译的字幕应直接复用缓存。以下情况才会重新调用 API：

- 该字幕 cue 从未成功翻译过；
- 修改了模型、服务商、Base URL、目标语言、源语言或 ASR 纠错开关；
- 手动清空了本地翻译缓存；
- 字幕合并算法版本发生变化。

## 项目结构

```
├── manifest.json          # Chrome MV3 扩展清单
├── src/
│   ├── background.js      # Service Worker：处理 API 调用和消息路由
│   ├── content.js         # Content Script：字幕获取、翻译队列、叠加层渲染
│   ├── shared.js          # 共享工具：设置、字幕合并、提示词生成
│   ├── overlay.css        # 字幕叠加层样式
│   └── page-bridge.js     # 页面级脚本，注入以拦截 YouTube 播放器数据
├── options/
│   ├── options.html       # 设置页面
│   ├── options.css        # 设置页面样式
│   └── options.js         # 设置页面逻辑
└── test/                  # 单元测试
```

## 开发

运行单元测试：

```powershell
node --test
```

如果 PowerShell 阻止运行 `npm.ps1`，可直接使用 `node --test` 或运行 `npm.cmd test`。
